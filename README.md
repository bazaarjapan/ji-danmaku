# 🎏 Ji-Danmaku

WindowsのPC画面とマイク入力を監視し、デスクトップ全面の透明オーバーレイに
**ニコニコ動画風の弾幕（視聴者コメント）** をリアルタイムで流すアプリです。
あたかも大勢の視聴者にライブ配信していて、画面に弾幕がドッと流れてくるような
爽快な体験ができます。

弾幕の「視聴者」は **Codex の app-server**（`codex app-server` / JSON-RPC）をブレインに、
いま画面に映っているスクリーンショットを見て反応コメントを生成します。マイクで喋ると
弾幕がドッと増えて盛り上がります。

> 💡 **追加料金なし**: ブレインは既存の `codex login`（ChatGPT / Codex サブスク）認証を
> そのまま使う app-server 方式です。API キーによる**従量課金は発生しません**。
> app-server プロセスは**1本だけ常駐**させて使い回すため軽量で、生成ごとに ephemeral
> スレッドを使って履歴を溜めない＝毎回の入力トークンを最小に保ちます。

---

## できること

- **画面全面オーバーレイ**：透明・最前面・クリック透過。弾幕は流れるだけで操作の邪魔をしません。
- **画面を見て反応**：定期的にスクショ＋前面ウィンドウ名を取り、Codex app-server がニコ動風コメントを生成。

- **マイク連動**：喋ると弾幕が増える／速くなる。
- **発話の理解（ローカルWhisper）**：マイク音声をこのPC上のWhisper（Transformers.js + onnxruntime-web）で
  文字起こしし、その内容をブレインに渡して弾幕に反映。**クラウド送信なし・追加課金なし**。
  初回だけモデルをDL（baseで数百MB）、以降はキャッシュからオフライン動作。
- **常に賑わう**：AIが無くても「アンビエント弾幕」(`www`/`草`/`888` 等)が流れ続けます。
- **3種のブレイン**：`codex`（推奨）/ `anthropic`（Claude Vision）/ `mock`（AI不要）。
  どれも失敗時は自動でアンビエントにフォールバックするので必ず動きます。
- **コントロール画面**：配信ON/OFF、生成間隔・速さ・文字サイズ・不透明度・密度を調整。

## 必要なもの

- Windows 10/11
- Node.js 18+（確認環境: v24）
- 推奨: [Codex CLI](https://github.com/openai/codex) がインストール＆ `codex login` 済み
  - 未ログイン/未インストールでもアプリは動きます（アンビエント弾幕にフォールバック）

## セットアップ & 起動

```powershell
npm install
npm start
```

起動すると **コントロール画面** と **透明オーバーレイ** が開きます。

1. コントロール画面の「▶ 配信スタート」（または **F8**）を押す
2. 普段どおりPCを使う → 画面の上を弾幕が流れ始めます
3. マイクにチェックを入れて喋ると、弾幕がドッと盛り上がります

## 配布用ビルド（Windows .exe）

[electron-builder](https://www.electron.build/) でインストーラ / ポータブル exe を生成できます。

```powershell
npm install        # 初回のみ
npm run dist
```

- 生成物は `dist/` に出力されます（`.gitignore` 済み）。
  - **インストーラ**: `Ji-Danmaku Setup <version>.exe`（NSIS / インストール先選択可）
  - **ポータブル**: `Ji-Danmaku <version>.exe`（インストール不要で起動）
- Whisper はレンダラーから `node_modules` を直接読むため、`asar` は無効化しています。
  未使用の重いネイティブ依存（`onnxruntime-node` / `sharp`）はビルドから除外してサイズを抑えています。
- アイコンは未同梱（Electron 既定）。`build.win.icon` に `.ico` を指定すると差し替えられます。

## 操作

| 操作 | 内容 |
|------|------|
| **F8** | 配信 ON / OFF（グローバル） |
| **F7** | コントロール画面を最前面に呼び出す（裏に隠れた時の救済） |
| コントロール画面 | ブレイン選択・各種スライダー・テスト弾幕 |
| テスト弾幕 | 任意の文字を即座に流して見た目を確認 |

## 設定

コントロール画面のスライダーで即時に変更できます。設定は
`%APPDATA%\ji-danmaku\config.json` に保存されます。

| 項目 | 説明 |
|------|------|
| ブレイン | `codex` / `anthropic` / `mock` |
| 生成間隔 | 画面を見て生成する周期（Codexは1回 ~20秒） |
| フィラー弾幕を追加 | メイン画面のチェック。OFFで **AIが生成した弾幕だけ** を流す（www/草/888 等の自動フィラー＆発話ざわめきを止める） |
| アンビエント密度 | フィラーON時に、AIの隙間で流れる自動弾幕の量（/分） |
| 流れる速さ / 文字サイズ / 不透明度 | 弾幕の見た目 |
| マルチモニター | `multiMonitor`。true で全ディスプレイに弾幕、false でプライマリのみ |
| キャプチャ対象 | `captureDisplayIndex`。AIに見せる画面の番号（null でプライマリ） |
| NGワード | `ngWords` / `ngMode`（drop=除外 / mask=伏字）。不適切語フィルタ |
| マイク監視 | 喋ると弾幕が増える（音量検知・常時オフライン） |
| 発話の文字起こし（Whisper） | ローカルWhisperで発話内容を認識しブレインへ渡す |
| Whisperモデル | `tiny`(最速/粗い) / `base`(推奨) / `small`(高精度/重い) |

### ローカルWhisper（音声認識）

マイク音声を**このPCのCPU/GPUだけ**で文字起こしします（Transformers.js + onnxruntime-web）。

- **完全ローカル・無料**：音声は外部に送信されず、API従量課金もありません。
- **初回のみDL**：選んだモデルを HuggingFace から取得し、以降はブラウザキャッシュから
  オフライン動作。`tiny`≈150MB / `base`≈数百MB。
- **バックエンド**：WebGPU が使える環境では自動でGPU実行（高速）、なければCPU(WASM)に
  自動フォールバック。コントロール画面に現在のバックエンドが表示されます。
- 発話の区切り（無音）を検知して、その単位で文字起こし→直近の認識結果を生成時の文脈に渡します。

### ブレインの切り替え

- **Codex**（既定 / app-server）: `codex login` 済みであればそのまま使えます。
  `codex app-server` を常駐させ、effort=low で高速・低コスト。**追加のAPI課金なし**。
- **Claude**: 環境変数 `ANTHROPIC_API_KEY` を設定すると Vision で画面に反応します。
  ```powershell
  $env:ANTHROPIC_API_KEY = "sk-ant-..."
  npm start
  ```
- **mock**: AI不要。文脈ワード＋定番リアクションだけで賑わいを作ります。

## 構成

```
electron/
  main.js        メインプロセス（ウィンドウ/キャプチャループ/IPC）
  preload.js     レンダラへの安全なAPIブリッジ
  config.js      設定の読み書き（%APPDATA%）
  screen.js      前面ウィンドウ名＋スクリーンショット取得
  ai/
    index.js     ブレインのディスパッチャ（失敗時 mock フォールバック）
    codex.js     Codex app-server 連携（常駐JSON-RPC / 画像入力 / effort=low / 課金なし）
    anthropic.js Claude Vision 連携（任意）
    mock.js      アンビエント弾幕生成
renderer/
  overlay.*          透明オーバーレイ＋弾幕エンジン（レーン管理/CSS合成）
  control.*          コントロール画面＋マイク監視（getUserMedia/VU/PCM収集/VAD）
  whisper-worker.js  ローカルWhisper(Transformers.js)を回すWeb Worker
```

## 仕組みのポイント

- **弾幕エンジン**：各コメントをレーン(行)に割り当て、右端→左端へ `transform` で
  GPU合成スクロール。衝突しないレーンを占有時刻ベースで選択し、重なりを最小化。
- **連続感**：AIバッチは一気に出さず、間隔内で少しずつ「ドリップ」放出。発話中はテンポUP。
- **負荷ガード**：同時表示数に上限。生成中は次サイクルを自動スキップしてプロセス重複を防止。

## 注意

- 画面キャプチャ・マイクを使用します。配信や録画と併用する際はプライバシーにご注意ください。
- Codex ブレインは `codex login`（サブスク）認証を使うため**従量課金は発生しません**。
  生成頻度はサブスクの利用上限の範囲内です。間隔は設定で調整できます。
